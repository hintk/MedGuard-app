# MedGuard 用药安全守护系统 — 作品说明文档

## 一、作品概述

**MedGuard** 是一款面向家庭用药安全管理的 iOS 原生应用。通过 **AI 大模型驱动的药物识别与风险分析**、**家庭成员双向绑定关怀**、以及 **6 位 PIN + Face ID 安全体系**，为老年人和子女提供一站式的用药管理、风险预警和亲情互动服务。

- **目标用户**：长期服药的老年人及其子女
- **核心价值**：用 AI 降低误服/漏服/药物冲突风险，用双向通知连接家庭关怀
- **运行平台**：iOS 16.0+（iPhone / iPad）

---

## 二、解决的问题

| 痛点 | 我们的解决方案 |
|------|---------------|
| 老年人看不清药盒，容易吃错药 | 📷 **拍照 AI 识别**：拍药盒自动提取药品名称、分类、规格 |
| 同时服用多种药物，有相互作用风险 | 🤖 **豆包大模型风险分析**：将所有在用药品发给 AI，输出高/中/低风险报告 |
| 子女不在身边，无法知晓父母用药情况 | 👨‍👩‍👧 **双向绑定 + 实时通知**：老人服药状态自动通知子女，子女可发送关怀回复 |
| 用药数据杂乱无章 | 📊 **智能档案 + 时间线**：按天分组，库存预警，用药历史一目了然 |
| 数据安全无保障 | 🔐 **6 位 PIN + Face ID + Keychain 加密存储** |

## 三、核心功能

### 3.1 智能用药管理

- **📋 药品档案**：完整记录药品名称、分类、规格、剂量、库存、来源
- **⏰ 用药时间线**：按「今天/昨天/更早」分组，支持标记服用/跳过
- **📦 库存预警**：自动计算剩余天数，低于阈值提示补充
- **🔍 条形码扫描**：扫药盒条码查药品信息（内置 30+ 常用药品数据库 + 百度 API 查询）

### 3.2 AI 智能分析（豆包大模型）

- **📷 拍照识别药品**：调用 **Doubao-Seed-1.6-vision** 视觉模型，拍照自动识别药盒
- **🧬 药物相互作用分析**：将所有在用药品完整信息发送给豆包 AI，由「资深临床药师」角色进行相互作用评估，输出高/中/低风险等级及详细建议
- **⚙️ 可更换 AI 模型**：设置页支持用户自行更换 API Key / Model / Endpoint，接入更高端模型

### 3.3 家庭双向关怀

- **🔗 6 位绑定码**：老人生成一次性绑定码（5 分钟有效），子女输入完成绑定
- **📬 实时通知系统**：老人服药/漏服自动推送通知给子女
- **💬 快捷回复**：老人可一键发送 👍🙏😊😟 表情回复子女
- **👥 多账号支持**：一部手机上可创建多个老人/子女账号

### 3.4 安全体系

- **🔢 6 位数字 PIN**：首次注册必须设置，SHA-256 加盐哈希存入 iOS Keychain
- **👤 Face ID / Touch ID**：支持面容/指纹生物识别解锁
- **🔑 PIN 保护设置**：修改 API 配置、变更安全设置均需 PIN 验证
- **🛡️ 数据本地存储**：所有数据仅存于设备本地（UserDefaults + Keychain），不上传服务器

## 四、技术架构

### 4.1 技术栈

| 层级 | 技术方案 |
|------|----------|
| UI 框架 | SwiftUI 4.0（iOS 16+） |
| 架构模式 | MVVM + Singleton Service Layer |
| 数据持久化 | UserDefaults (JSON Codable) + iOS Keychain |
| AI 服务 | 豆包大模型 (Doubao-Seed-1.6-vision + Doubao-Lite) via Volces Ark API |
| 生物识别 | LocalAuthentication (Face ID / Touch ID) |
| 通知系统 | UNUserNotificationCenter (本地通知) |
| 条形码查询 | 本地 DrugDatabase + 百度 API |

### 4.2 项目结构

```
MedGuard/
├── App/
│   ├── MedGuardApp.swift          # @main 入口 + AppDelegate
│   └── MainTabView.swift          # 5-Tab 主框架
├── Stores/
│   ├── AuthStore.swift            # 多用户认证、绑定码、通知记录
│   ├── PinStore.swift             # 6位PIN + Keychain + SHA-256
│   ├── APIConfigStore.swift       # AI API 配置持久化
│   └── MedicationStore.swift      # 药品增删改查 + 时间线
├── Services/
│   ├── DoubaoService.swift        # 豆包大模型（视觉识别 + 风险分析）
│   ├── RiskEngine.swift           # 本地规则引擎（10条交互规则）
│   ├── DrugLookupService.swift    # 条形码查询
│   ├── DrugDatabase.swift         # 内置药品数据库（30+药品）
│   ├── NotificationManager.swift  # 通知管理

│   ├── Home/                      # 首页 / 档案 / 添加药品
```

### 4.3 数据流

```
用户操作 → View (SwiftUI)
           ↓ @EnvironmentObject / @StateObject
       Store (ObservableObject)
           ↓ async/await
       Service (Singleton)
           ↓ URLSession / UserDefaults / Keychain
       外部 API / 本地存储
```

### 4.4 安全设计

- **PIN 存储**：SHA-256 哈希 + 16 字节随机盐 → iOS Keychain（`kSecClassGenericPassword`）
- **防暴力破解**：5 次错误尝试后锁定 30 秒
- **API Key**：存储于 UserDefaults，可随时更换，支持自定义模型接入
- **数据隔离**：每个用户的用药数据和通知记录按 `userId` 隔离

## 六、创新亮点

1. **大模型 + 规则引擎双重保障**：豆包 AI 提供深度分析，本地 RiskEngine（10 条临床规则）作为离线兜底
2. **AI 模型可替换**：不绑定单一模型厂商，用户可接入任何 OpenAI-compatible API
3. **家庭双向关怀闭环**：老人服药 → 自动通知子女 → 子女查看 → 老人收到关怀回复
4. **安全无感**：6 位 PIN 首次必设，日常 Face ID 无感解锁，Keychain 硬件级加密
5. **全本地存储**：用户数据不上传云端，隐私 100% 掌控
